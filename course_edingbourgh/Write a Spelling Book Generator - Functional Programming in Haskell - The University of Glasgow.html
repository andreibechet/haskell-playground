Write a Spelling Book Generator

37 comments

Are you familiar with children’s alphabetical spelling books? They say things like: a is for apple, b is for baby, and c is for cat

Now that you know about Haskell list functions, you can develop a function to generate the text for a spelling book, given a list of words.

Suppose the function is called speller then it should have the following type:

speller :: [[Char]] -> [Char]

recalling that a list of Char elements is a String in Haskell.

Typical executions of the speller function would look like this:

speller ["abacus"] -- > "a is for abacus"
speller [] -- > ""
speller ["apple", "banana", "coconut"] 
 -- > "a is for apple, b is for banana, and c is for coconut"
speller ["whisky", "x-ray"]
 -- > "w is for whisky, and x is for x-ray"

Your task is to develop the Haskell speller function. You are allowed to define small helper functions, perhaps to generate a single letter’s phrase from a word: f "zoo" -- > "z is for zoo". You are also allowed to use the standard Haskell list functions like map and foldr if appropriate.

Write your Haskell source code in a .hs file, and copy/paste it into the exercise activity below. At the end of this week, we will ask learners to peer-review submitted code anonymously, by comparing it with other submissions.

When you write your code, please pay attention to:

  • type declarations for functions
  • pattern matching for different list structures (i.e. empty list, single element list, multi-element list)
  • neat formatting of code
  • concise, explanatory comments

    Better late than never!

    import Data.Char

    speller :: [[Char]] -> [Char]
    speller [] = []
    speller [x] = spellOut x ++ "."
    speller (x:y:[]) = spellOut x ++ ", and " ++ speller [y]
    speller (x:xs) = spellOut x ++ ", " ++ speller xs

    spellOut :: [Char] -> [Char]
    spellOut (x:xs) = [toUpper x] ++ " is for " ++ x:xs

    Here I have 4 recursive cases: speller [x] binds when there is only one item x in the list, so adds a "." to the end of the string.

    speller (x:y:[]) fires when there are only two items remaining in the list (x and y), adding ", and" between the second last and last items. The notation here is a bit odd to me, but similarly if you wanted to bind with exactly three remaining elements you might write something like (x:y:z:[])

    Note that I've also used the function toUpper from Data.Char to uppercase the letter.

    speller :: [String] -> String
    speller [] = ""
    speller words = if null xs
    then letterOf x
    else join (map letterOf (reverse xs)) ++ "and " ++ letterOf x
    where (x : xs) = reverse words

    letterOf :: String -> String
    letterOf word = [head word] ++ " is for " ++ word

    join :: [String] -> String
    join = foldr ((++) . (++ ", ")) ""

    -- speller.hs

    wordToSentence :: [Char] -> [Char]
    wordToSentence word
    | null word = ""
    | otherwise = [x] ++ " is for " ++ word
    where x:_=word

    concat_sentences :: [[Char]] -> [Char]
    concat_sentences lst
    | null lst = []
    | length lst == 1 = ", and " ++ x
    | otherwise = ", " ++ x ++ concat_sentences xs
    where x:xs=lst

    speller :: [[Char]] -> [Char]
    speller words
    | null words = []
    | length words == 1 = wordToSentence word
    | otherwise = wordToSentence word ++ concat_sentences (fmap wordToSentence remainingWords)
    where word:remainingWords=words
